** DONE tblfn-aggregateで複数の列を集計できるようにする
CLOSED: [2026-05-24 Sun 17:41]
現在は個別に集計して結合しなければならない。
tblfn-aggregate-columnsのような関数を追加することもできるが、VALUE-COLSPEC-OR-FUNCTIONに複数指定できるようにした方が使いやすい気もする。

VALUE-COLSPEC-OR-FUNCTIONは現状では:
- COLSPEC
  - integer
  - string
- FUNCTION
  - S-expression (最初の要素が関数)
  - Function

なのでこれを複数にすることは一応はできる:
: ( INTEGER|STRING +|SEXP|FUNCTION+ |(INTEGER|STRING|SEXP|FUNCTION [VFUN [COLNAME]] [:calcopt CALCOPT] ) ... )

リストの先頭がintegerp、stringp、conspであるなら複数列指定、functionpなら従来のS式と判定する。

その場合、VFUN、NEW-VALUE-COLNAME、CALCOPT引数が無駄になる。省略時のデフォルト値として活用しても良い。

CLOSED: [2026-06-22 Mon 16:18]

関数名はtblfn-trade-remaining-quantity-and-costなど色々と候補を出したが、最終的にtblfn-trade-resultにした。何かより一般的な概念があるか探したけれど今ひとつ良い言葉が見つからないので、取引に特化した用語を使うことにする。もちろん他の用途にも十分使えると思う。在庫管理とか。

本質的には、どこか特定の場所に何かを出し入れする際に使えるはず。極端な話、塩分濃度が異なる塩水を出し入れするような場合の計算にも使えるかもしれない。

戻り値は (quantity cost average-cost profit) という形式のリストとする。テーブルにしたいなら別途listで囲めば良い。

派生的な関数も含めて次の関数を用意する。
- tblfn-trade-result
- tblfn-trade-result-quantity
- tblfn-trade-result-cost
- tblfn-trade-result-average-cost
- tblfn-trade-result-profit

tblfn-trade-resultがあれば全て実装できるが、tblfn-trade-result-costは平均処理が必要ないのでより効率が良い実装を使っても良い。

取引のたびに値を補正する機能が欲しい。取引の後で端数処理が必要になる場合がある。
cost-adjust-funとprofit-adjust-fun引数を追加。

結果の値をどのような型で返すべきか。math関数の結果(Calcの内部表現)で返すのも能が無い。ほとんどの用途では文字列がベストだろう。tblfn-math-convert-valueという関数を追加してresult-type引数で指定できるようにする。calcopt引数も追加して桁数などを制御できるようにする。リストを指定することで結果リストの要素毎に指定出来るようにする。

投資信託では10000口あたりの平均取得価額が欲しい。米ドルMMFなら100口単位で1ドル単位。average-cost-unit引数を追加。
